Quick debugging checklist:

  • Run the app locally with debugging enabled to see the full traceback (set FLASK_ENV=development and start Flask).
  • Check Vercel / deployment logs for the traceback (Vercel Dashboard → Deployments → View Logs or use vercel logs).
  • Look for missing templates, missing endpoints, import errors, or database exceptions in the traceback and fix them in the corresponding view.

If you need the traceback locally, run these commands in your project root:

# Windows (cmd) set FLASK_APP=app.py
set FLASK_ENV=development
flask run

After fixing, commit and push to GitHub and Vercel will auto-deploy:

git add -A
git commit -m "Fix server error"
git push origin main
